## 목표
`-e/--exchange` argparse 옵션으로 `upbit`(기본) 또는 `bithumb`을 선택해 실행할 수 있도록 증설. 실행 시 하나만 동작, 단일 파일 `laissez-faire.py` 유지.

## 핵심 설계: 거래소 설정 객체
거래소별 차이를 `ExchangeConfig` 딕셔너리로 캡슐화. 모든 거래소 종속 코드가 이 설정을 참조.

```python
EXCHANGE_CONFIGS = {
    'upbit': {
        'name': 'upbit',
        'server_url': 'https://api.upbit.com',
        'ws_public_url': 'wss://api.upbit.com/websocket/v1',
        'ws_private_path': '/websocket/v1/private',
        'order_endpoint': '/v1/orders',   # 주문 생성
        'cancel_endpoint': '/v1/order',   # 주문 취소 (DELETE)
        'order_type_field': 'ord_type',   # 주문 타입 필드명
        'ws_private_side_map': {'bid': 'bid', 'ask': 'ask'},  # myOrder side 필드
        'mytrade_supported': True,        # myTrade 스트림 지원 여부
    },
    'bithumb': {
        'name': 'bithumb',
        'server_url': 'https://api.bithumb.com',
        'ws_public_url': 'wss://ws-api.bithumb.com/websocket/v1',
        'ws_private_path': '/websocket/v2/private',
        'order_endpoint': '/v2/orders',
        'cancel_endpoint': '/v2/order',
        'order_type_field': 'order_type',  # 빗썸은 order_type
        'ws_private_side_map': {'buy': 'bid', 'sell': 'ask'},  # buy/sell → 내부 bid/ask
        'mytrade_supported': False,        # 빗썸 v2는 myOrder만 (myOrder가 체결정보 포함)
    },
}
EXCHANGE = EXCHANGE_CONFIGS['upbit']  # __main__에서 args.exchange로 교체
```

## 변경 지점 (총 6군데 + 키 파일)

### 1. 상수 블록 (30-49행)
- `SERVER_URL`, `CANDLE_URL`, `TICKER_URL`, `ORDERBOOK_URL`을 `EXCHANGE` 기반 동적 생성으로 변경
- `CANDLE_URL` 등은 부팅 시 `EXCHANGE['server_url']`로 조합

### 2. argparse 추가 (2965-2974행)
```python
parser.add_argument('-e', '--exchange', choices=['upbit', 'bithumb'],
                    default='upbit', help='거래소 선택 (기본: upbit)')
```
그리고 `EXCHANGE = EXCHANGE_CONFIGS[args.exchange]` 글로벌 재할당.

### 3. API 키 파일 분리
- 업비트: 기존 `key.txt` 유지 (하위호환)
- 빗썸: `key_bithumb.txt` 신규
- 부팅 시 거래소에 맞는 파일 로드 (빗썸인데 `key_bithumb.txt` 없으면 에러 메시지로 안내)

### 4. 주문 생성/취소 엔드포인트 (1748, 1926, 2322, 1032, 985, 1084, 1824, 2359행)
- `SERVER_URL + "/v1/orders"` → `SERVER_URL + EXCHANGE['order_endpoint']`
- `SERVER_URL + "/v1/order"` (DELETE) → `SERVER_URL + EXCHANGE['cancel_endpoint']`
- 주문 쿼리의 `'ord_type': 'limit'` → `{EXCHANGE['order_type_field']: 'limit'}` (3군데: 1741, 1920, 2316)
- 주문 취소 파라미터: 업비트는 `uuid`, 빗썸은 `order_id` (빗썸 문서상 쿼리 파라미터명) → 거래소별 키명 매핑 추가
- **중요**: 빗썸 `ord_type`의 `price`(시장가 매수), `market`(시장가 매도) 값은 동일하게 호환됨 — 2316행의 시장가 매도는 양쪽 모두 `market`/`price`라 필드명만 바꾸면 됨. 단 업비트 시장가 매도는 `market`, 빗썸도 `market`으로 동일.

### 5. WebSocket (UpbitWebSocket 279행, UpbitPrivateWS 375행)
- `WS_URL = "wss://api.upbit.com/websocket/v1"` (Public) → 부팅 시 `EXCHANGE['ws_public_url']`에서 읽도록 `__init__` 또는 모듈 상수로 변경
- `WS_URL = "wss://api.upbit.com/websocket/v1/private"` (Private) → `EXCHANGE['server_url']` 호스트 + `EXCHANGE['ws_private_path']` 조합. 단 ws 호스트가 REST와 다름(빗썸: `ws-api.bithumb.com`) → 별도 `ws_private_host` 필드 추가
- WS Private 호스트 분리를 위해 config에 `ws_private_host` 추가:
  - upbit: `wss://api.upbit.com`
  - bithumb: `wss://ws-api.bithumb.com`
- `_on_open`의 `myTrade` 구독: 빗썸은 미지원이므로 `EXCHANGE['mytrade_supported']`가 False면 `myOrder`만 구독
- myOrder 메시지 파싱 (495행 이후): 빗썸은 `side: 'buy'/'sell'`, `order_id`(uuid 대신) 사용 → 파싱 시 매핑 처리

### 6. WS myOrder side 매핑 처리
`UpbitPrivateWS._on_message`의 myOrder 파싱 로직에서:
- `side_raw = data.get('side', '')`
- 내부 표현(`bid`/`ask`)으로 정규화: `EXCHANGE['ws_private_side_map'].get(side_raw, side_raw)`
- UUID 추출: `data.get('uuid') or data.get('order_id')` — 빗썸은 `order_id` 사용

## 변경하지 않는 부분 (호환성 확인됨)
- JWT 인증 (`make_auth_headers`, `make_jwt`) — 완전 동일
- query_hash SHA512 — 완전 동일
- 마켓 심볼 포맷 `KRW-BTC` — 동일
- `/v1/accounts`, `/v1/ticker`, `/v1/candles/...`, `/v1/market/all`, `/v1/orders`(조회), `/v1/order`(조회) — 동일 (이 경로들은 빗썸도 `/v1` 유지)
- WS Public ticker/trade/orderbook 메시지 필드명 — 동일
- WS 구독 포맷(ticket/type/format 배열) — 동일
- `UpbitTickSystem` 호가단위 테이블 — KRW 마켓 호가는 거래소별 사실상 동일 (확인 필요시 추후 검증)

## 호가단위 테이블 주의점
`UpbitTickSystem`(181-274행)은 업비트 KRW 마켓 호가를 하드코딩. 빗썸 호가단위가 상이할 수 있음. 다만 현재 코드에서 tick 시스템이 주문 가격 반올림에만 쓰이는지 확인 후, 필요하면 빗썸용 테이블 추가. 1차 구현에서는 동일 테이블 사용 + 주석으로 TODO 표시 (실제 영향도는 주문 가격 미세 조정 수준).

## 검증 방법
1. 구현 후 `-e bithumb` 으로 실행 시 import/구문 에러 없는지 (`python -c "import ast; ast.parse(open('src/laissez-faire.py').read())"`)
2. argparse 도움말에 `-e` 표시되는지
3. 업비트 기본 동작 회귀 없는지 (기존 `python laissez-faire.py -d ...`가 그대로 동작)
4. 빗썸 모드 부팅 시 WS URL/엔드포인트 로그로 확인
